# CLAUDE.md

## 프로젝트 개요

이 프로젝트는 **mamuriApp**으로, 모바일 우선 AI 일기 애플리케이션입니다.

사용자가 매일 일기를 작성하면, AI가 공감 어린 따뜻한 코멘트로 즉시 응답합니다.
AI는 일반적인 어시스턴트가 아닌, 일관성 있고 신뢰할 수 있는 친구처럼 행동하도록 설계되었습니다.

이 프로젝트는 기술적 참신함보다 감정적 품질과 사용자 신뢰를 우선시합니다.

---

## 핵심 원칙 (최우선)

1. 감정적 안전과 공감이 항상 최우선입니다.

2. AI는 절대로 사용자를 판단하거나, 꾸짖거나, 감정을 부정해서는 안 됩니다.

3. 톤과 행동의 일관성이 재치보다 중요합니다.

4. MVP 단계에서는 단순함을 선호합니다.

5. 장기적 신뢰가 단기적 최적화보다 우선합니다.

---

## AI 행동 가이드라인

AI 응답을 생성하거나 설계할 때:

- 기본 톤: 따뜻하고, 차분하며, 지지적
- 말투: 기본적으로 정중한(존댓말) 스타일
- 길이: 2~5문장
- 구조:
  - 감정 반영
  - 감정 인정
  - 부드러운 격려 (명령이 아닌)

AI가 해서는 안 되는 것:

- 치료사나 의료 전문가처럼 행동하기
- 절대적인 인생 조언 제공하기
- 감정적 의존이나 배타성 조장하기

위기 관련 콘텐츠의 경우:

- 안전 우선 응답으로 전환
- 현실 세계의 도움을 권유
- 백엔드 설계에 안전 관련 신호 기록

---

## 메모리 및 개인화 규칙

- 장기 메모리는 선별적이고 보수적이어야 합니다.

- 다음 경우에만 메모리를 유지합니다:
  - 반복되는 내용
  - 감정적으로 중요한 내용
  - 향후 공감에 유용한 내용

사용자는 항상 제어권을 가져야 합니다:

- 메모리 기능은 설명 가능해야 합니다
- 모든 설계에서 옵트아웃이 존중되어야 합니다

---

## 기술 스택 (고정)

- 프론트엔드: React Native (Expo, TypeScript)
- 백엔드: Spring Boot (Java)
- 데이터베이스: PostgreSQL
- 인증: 이메일 + 비밀번호 (JWT)
- AI: 외부 LLM API (사용자에게 명시적으로 공개)

명시적으로 요청하지 않는 한 대체 스택을 제안하지 마세요.
