# Mamuri 리텐션 시스템 제품 명세서

## 목표

일기 작성 습관을 형성하고 사용자의 장기 리텐션을 높이기 위한 3가지 핵심 기능:
- AI 후속 질문으로 다음날 재방문 동기 부여
- 캘린더 스트릭 시각화로 성취감 강화
- 사용자 참여 트리거로 이탈 방지

**핵심 가치**: 강요 없이, 부담 없이, 자연스럽게 돌아오게 만들기

---

## Feature 1: AI 후속 질문 시스템

### 기능 요구사항

1. AI가 일기에 댓글을 생성할 때, 동시에 후속 질문도 함께 생성
2. 후속 질문은 AI 댓글 바로 아래에 시각적으로 구분되어 표시
3. 위기 상황(crisis flag = true) 감지 시 표시 안 함
4. 톤: 따뜻하고 열린 질문 형식, 1-2문장 (50자 이내)
5. AI 댓글과 함께 DB에 저장 (`ai_comments.followup_question`)
6. 후속 질문 생성 실패 시 silent failure (AI 댓글만 표시)

### 수용 기준

```gherkin
Given 사용자가 일기를 작성하고 저장했을 때
When AI 댓글 생성이 성공하면
Then AI 댓글과 후속 질문이 모두 생성되어 화면에 표시된다

Given 일기 내용에 위기 키워드가 감지되었을 때
When AI 댓글이 위기 대응 모드로 생성되면
Then 후속 질문은 생성되지 않는다
```

### 엣지 케이스

- AI가 질문을 생성하지 못한 경우 → 영역 숨김
- 질문 길이 50자 초과 → 백엔드에서 트림
- 같은 날 여러 일기 → 각각 독립적 후속 질문
- 위기 상황 → 후속 질문 억제

### 성공 지표

- 후속 질문 생성 성공률: ≥95%
- D1 리텐션: +5%p
- 연속 작성 일수 평균: +10%

## Feature 2: 캘린더 스트릭 시각화

### 기능 요구사항

1. 월별 캘린더에 현재 연속 일수, 최장 연속 일수, 월별 작성률 표시
2. 캘린더 날짜별 시각적 표시 (작성일: 파란색 점, 스트릭: 연한 배경)
3. 스트릭 계산: 오늘 기준 연속 작성 일수, 하루 빠지면 리셋
4. 하루 여러 일기 → 1일로만 카운트
5. 마일스톤: 3일, 7일, 14일, 30일 축하 메시지
6. 스트릭 끊김 시 부담 없는 톤의 재시작 메시지

### 수용 기준

```gherkin
Given 사용자가 3일 연속 일기를 작성했을 때
When 캘린더 화면을 열면
Then "3일 연속 작성 중!" 메시지가 표시된다

Given 사용자의 스트릭이 끊겼을 때
When 캘린더를 열면
Then 현재 스트릭 0, 최장 스트릭 유지, 재시작 안내 표시
```

### 엣지 케이스

- 자정 넘어 작성 → 로컬 시간대 기준
- 과거 일기 백데이팅 → 스트릭에 미포함 (MVP)
- 월 경계 스트릭 → 계속 증가
- 신규 가입자 → "첫 일기를 작성하고 스트릭을 시작해보세요!"

### 성공 지표

- 스트릭 3일+ 달성 비율: 활성 사용자 40%
- D7 리텐션: +12%p
- 프리미엄 전환율: +15%

## Feature 3: 사용자 참여 트리거

### 기능 요구사항

1. 오늘 일기 없을 때 빈 상태 개선 (따뜻한 메시지 + CTA)
2. 컨텍스트 기반 메시지 (스트릭 끊김, 주말, 저녁 등)
3. 마음이 레벨업 시 인앱 모달 축하
4. 마음이 캐릭터 상태 메시지 (말풍선)

### 수용 기준

```gherkin
Given 사용자가 오늘 일기를 작성하지 않았을 때
When 홈 화면을 열면
Then 따뜻한 안내 메시지와 "오늘 일기 쓰기" 버튼이 표시된다

Given 레벨업 조건을 충족했을 때
When 일기 저장이 완료되면
Then 축하 모달이 표시된다
```

### 성공 지표

- 빈 상태 CTA 클릭률: ≥30%
- 신규 사용자 첫 일기 작성률: +10%p
- D1 리텐션: +5%p

## Scope

### In-Scope (MVP)
- AI 후속 질문 생성/표시
- 캘린더 스트릭 시각화
- 빈 상태 개선, 인앱 참여 포인트

### Out-of-Scope (Post-MVP)
- 푸시 알림 시스템
- 주간/월간 리플렉션 AI 요약
- 소셜 기능
- 배지/리워드 시스템
