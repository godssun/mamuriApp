# AI Companion 개인화 시스템 제품 사양서

## 1. Feature Goal

**목표**: 사용자가 AI 친구("마음이")를 자신만의 친구로 느낄 수 있도록 개인화 옵션을 제공하여 감정적 연결과 신뢰를 강화한다.

**핵심 가치**:
- 단순한 AI 댓글이 아닌, "나만의 친구"라는 소유감과 애착 형성
- 개인화를 통한 AI 응답 품질 향상 및 사용자 만족도 증대
- 프리미엄 기능으로 확장 가능한 기반 마련 (아바타, 말투 등)

**MVP 범위**: 기존 시스템(이름 변경, tone) 확장 + 아바타 선택 + 말투 스타일 추가

---

## 2. User Scenarios

### 시나리오 1: 아바타 선택으로 친근감 형성
```
사용자 상황: 신규 가입 후 AI 친구 화면에 처음 진입
행동: 기본 식물 이모지(🌰) 대신 자신이 좋아하는 동물(🐰)을 선택
결과: "이제 토끼가 나의 친구네!"라는 소유감과 애착 형성
```

### 시나리오 2: 말투 스타일로 AI 응답 커스터마이징
```
사용자 상황: 공식적인 존댓말보다 친근한 반말을 선호
행동: 설정에서 말투를 "캐주얼"로 변경
결과: AI 댓글이 "오늘 하루 힘들었겠다. 괜찮아, 잘 견뎠어!"로 바뀜
```

### 시나리오 3: 성격 톤 조정으로 개인 맞춤 응답
```
사용자 상황: 따뜻한 위로보다 현실적이고 담백한 응답을 선호
행동: 성격 톤을 "현실적"으로 설정
결과: "힘든 일이었네요. 잠시 쉬어가도 괜찮아요." 대신
       "힘든 하루였네요. 내일은 오늘보다 나을 거예요." 같은 응답 제공
```

### 시나리오 4: 이름 변경으로 개인적 연결
```
사용자 상황: 기본 이름 "마음이"보다 자신만의 이름을 짓고 싶음
행동: AI 친구 화면에서 "별이"로 이름 변경
결과: 모든 AI 응답에서 "별이가 응원할게요" 같은 문장으로 개인화
```

### 시나리오 5: 복합 설정으로 완전 개인화
```
사용자 상황: 자신만의 AI 친구를 완전히 커스터마이징하고 싶음
행동:
  - 아바타: 고양이 🐱
  - 이름: "나비"
  - 말투: 캐주얼
  - 성격: 밝고 긍정적
결과: "나비"라는 이름의 고양이 친구가 친근한 반말로 밝은 응원을 보냄
```

## 3. In-Scope (MVP)

### 필수 구현 항목

1. **아바타 선택 시스템**
   - 기본 제공 이모지 세트: 식물(🌰🌱🌿🌳), 동물(🐰🐱🐶🐻), 자연(☀️🌙⭐🌈)
   - 총 12개 아바타 옵션
   - 아바타 변경 UI (CompanionScreen)
   - UserSettings에 `avatar` 필드 추가

2. **말투 스타일 설정**
   - 포멀(존댓말, 기본값) vs. 캐주얼(반말)
   - UserSettings에 `speechStyle` 필드 추가
   - PromptBuilder에 말투 지시 문구 추가

3. **성격 톤 확장**
   - 기존 3가지(warm/calm/cheerful) → 4가지로 확장
   - 신규 추가: `realistic` (현실적이고 담백한)
   - 기존 `aiTone` 필드 재사용

4. **설정 화면 UI 개선**
   - 아바타 선택 섹션
   - 말투 스타일 토글
   - 성격 톤 라디오 버튼 (4가지)
   - 실시간 프리뷰 예시 문구

5. **API 확장**
   - `PUT /api/companion/avatar` (아바타 변경)
   - `PUT /api/settings/speech-style` (말투 변경)
   - 기존 `/api/settings/ai-tone` 확장 (realistic 추가)

### 기능 제약 조건
- 아바타는 이모지만 제공 (커스텀 이미지 업로드 불가)
- 말투는 2가지만 제공 (포멀/캐주얼)
- 성격 톤은 4가지로 제한
- AI 성장 시스템과 독립적 (레벨은 기존 시스템 유지)

## 4. Out-of-Scope (MVP 제외)

### 향후 확장 항목
1. **커스텀 아바타 업로드** (Post-MVP)
2. **AI 성장 시스템 고도화** (현재 feature flag로 비활성화 유지)
   - 레벨별 능력 해금
   - 성장 스토리라인
   - 아바타 진화/변신
3. **추가 말투 옵션** (예: 문어체, 시적 표현 등)
4. **감정 상태 기반 응답** (사용자 감정 패턴 학습)
5. **프리미엄 아바타 팩** (수익화 연계)
6. **음성 톤 설정** (TTS 통합 시)
7. **AI 성격 퀴즈** (추천 설정)

### 명시적 제외 항목
- AI 행동 패턴 학습 (너무 복잡, MVP 아님)
- 사용자 생성 아바타 (보안/저작권 이슈)
- 다중 AI 친구 (혼란 유발, 핵심 가치 훼손)

## 5. Functional Requirements

### FR-1: 아바타 선택 시스템
- **FR-1.1**: 사용자는 12개의 사전 정의된 아바타 이모지 중 하나를 선택할 수 있다
- **FR-1.2**: 선택한 아바타는 CompanionScreen 프로필 카드에 표시된다
- **FR-1.3**: 아바타 변경은 즉시 반영되며, 일기 상세 화면의 AI 댓글 영역에도 표시된다
- **FR-1.4**: 기본 아바타는 Lv.1 식물(🌰)이다

### FR-2: 말투 스타일 설정
- **FR-2.1**: 사용자는 포멀(존댓말) 또는 캐주얼(반말) 중 선택할 수 있다
- **FR-2.2**: 기본값은 포멀(존댓말)이다
- **FR-2.3**: 말투 변경 시 프롬프트에 다음 문구 추가:
  - 포멀: "존댓말을 사용하여 정중하게 이야기해 줘"
  - 캐주얼: "친근한 반말로 편하게 이야기해 줘"
- **FR-2.4**: 말투 변경은 다음 일기부터 적용된다 (기존 AI 댓글 유지)

### FR-3: 성격 톤 확장
- **FR-3.1**: 기존 3가지 톤(warm/calm/cheerful) + realistic 추가
- **FR-3.2**: `realistic` 톤 프롬프트 문구: "현실적이고 담백하게, 과하지 않은 격려로 이야기해 줘"
- **FR-3.3**: 톤 변경은 즉시 적용되며, 다음 일기 AI 댓글부터 반영된다

### FR-4: 설정 UI 개선
- **FR-4.1**: CompanionScreen에 "아바타 변경" 버튼 추가
- **FR-4.2**: 설정 화면에 "말투 스타일" 토글 추가
- **FR-4.3**: 설정 화면에 "성격 톤" 라디오 버튼 4개 추가
- **FR-4.4**: 각 설정 변경 시 예시 문구 표시:
  - 예: "포멀 + warm" → "오늘 하루 많이 힘드셨겠어요. 충분히 쉬세요."
  - 예: "캐주얼 + cheerful" → "오늘 고생 많았어! 내일은 더 좋을 거야!"

### FR-5: API 확장
- **FR-5.1**: `PUT /api/companion/avatar { "avatar": "🐰" }` 엔드포인트 추가
- **FR-5.2**: `PUT /api/settings/speech-style { "speechStyle": "casual" }` 엔드포인트 추가
- **FR-5.3**: 기존 `/api/settings/ai-tone` 요청 body validation에 "realistic" 추가

### FR-6: 프롬프트 조립 로직
- **FR-6.1**: PromptBuilder에 `SPEECH_STYLE_MAP` 추가
- **FR-6.2**: 프롬프트 템플릿에 `{{speechStyleInstruction}}` 변수 추가
- **FR-6.3**: 말투 + 톤 조합이 자연스럽게 융합되도록 프롬프트 구조 개선

## 6. Non-Functional Requirements

### NFR-1: 성능
- 아바타 변경 API 응답 시간: < 200ms
- 설정 변경 후 UI 반영: < 100ms (즉시 피드백)
- 프롬프트 조립 오버헤드: < 50ms 추가

### NFR-2: 사용성
- 아바타 선택 UI는 3탭 이내 접근 가능
- 설정 변경 시 명확한 성공 피드백 제공
- 각 설정 옵션마다 설명 툴팁 제공

### NFR-3: 확장성
- 아바타 이모지 추가 시 코드 변경 최소화 (설정 파일 관리)
- 새로운 말투/톤 추가 가능한 구조
- 프리미엄 기능 확장 가능한 DB 스키마

### NFR-4: 데이터 정합성
- 아바타 값 검증 (허용된 이모지만 저장)
- 말투/톤 값 enum validation
- 기본값 fallback 보장

### NFR-5: 일관성
- 모든 화면에서 동일한 아바타 표시
- 설정 변경이 즉시 CompanionScreen에 반영
- AI 댓글 톤 일관성 유지

## 7. Monetization Impact

### 직접적 영향
- **프리미엄 아바타 팩** (Post-MVP): 유료 아바타 추가 시 수익 발생 가능
- **개인화 옵션 제한**: 무료 사용자는 기본 아바타 3개, 프리미엄은 전체 12개 접근
  - MVP에서는 전체 무료 제공, 추후 수익화 전환 가능

### 간접적 영향
- **리텐션 향상**: 개인화로 인한 애착 형성 → D7 리텐션 +8%p 예상
- **프리미엄 전환**: "더 많은 아바타"가 프리미엄 유인 요소 → 전환율 +5%p
- **바이럴 효과**: "나만의 AI 친구" 컨셉이 SNS 공유 촉진

### 수익화 가설
- 개인화 만족도 ↑ → 장기 사용자 ↑ → 프리미엄 전환율 ↑
- 아바타/톤 조합으로 "나만의 친구" 느낌 → 이탈률 ↓

## 8. Analytics Events Required

### 신규 이벤트
```typescript
// 아바타 변경
companion_avatar_changed: {
  user_id: string,
  old_avatar: string,
  new_avatar: string,
  timestamp: DateTime
}

// 말투 변경
companion_speech_style_changed: {
  user_id: string,
  old_style: 'formal' | 'casual',
  new_style: 'formal' | 'casual',
  timestamp: DateTime
}

// 성격 톤 변경
companion_tone_changed: {
  user_id: string,
  old_tone: string,
  new_tone: string,
  timestamp: DateTime
}

// 설정 화면 진입
companion_settings_opened: {
  user_id: string,
  source: 'companion_screen' | 'settings_screen',
  timestamp: DateTime
}

// 아바타 선택 모달 열림
companion_avatar_modal_opened: {
  user_id: string,
  current_avatar: string,
  timestamp: DateTime
}
```

### 기존 이벤트 확장
```typescript
// AI 댓글 생성 시 설정 정보 추가
ai_comment_generated: {
  ...,
  avatar: string,
  speech_style: string,
  tone: string
}
```

### 분석 지표
- 아바타 변경 빈도 (사용자당 평균)
- 가장 인기 있는 아바타 TOP 3
- 말투 선택 분포 (포멀 vs. 캐주얼)
- 톤 선택 분포 (4가지)
- 설정 변경 후 리텐션 변화

## 9. Retention Hypothesis

### 가설 1: 개인화가 애착을 형성한다
- **측정**: 아바타/이름 변경 사용자 vs. 미변경 사용자 D7 리텐션 비교
- **예상**: 개인화 사용자 D7 리텐션 +10%p

### 가설 2: 말투 선택이 AI 만족도를 높인다
- **측정**: 말투 설정 사용자의 AI 댓글 재시도율 감소
- **예상**: 캐주얼 설정 사용자 재시도율 -15%

### 가설 3: 복합 개인화가 프리미엄 전환을 촉진한다
- **측정**: 3가지 이상 설정 변경 사용자의 프리미엄 전환율
- **예상**: 복합 개인화 사용자 전환율 +12%p

### 검증 방법
- A/B 테스트: 개인화 옵션 노출 vs. 미노출 그룹
- 코호트 분석: 설정 변경 시점 기준 전후 리텐션 비교
- 사용자 인터뷰: 개인화 만족도 정성 조사

## 10. Acceptance Criteria (Given/When/Then)

### AC-1: 아바타 선택
```gherkin
Given 사용자가 CompanionScreen에 진입했을 때
When "아바타 변경" 버튼을 탭하면
Then 12개의 아바타 선택 모달이 표시된다

Given 아바타 선택 모달에서 새 아바타를 선택했을 때
When "저장" 버튼을 탭하면
Then 프로필 카드의 이모지가 즉시 변경된다
And 성공 토스트 메시지가 표시된다
```

### AC-2: 말투 스타일 변경
```gherkin
Given 사용자가 설정 화면에서 말투를 "캐주얼"로 선택했을 때
When 새로운 일기를 작성하면
Then AI 댓글이 반말 톤으로 생성된다
And 기존 일기의 AI 댓글은 변경되지 않는다
```

### AC-3: 성격 톤 변경
```gherkin
Given 사용자가 설정 화면에서 톤을 "realistic"로 선택했을 때
When 예시 문구 영역을 확인하면
Then "현실적이고 담백한" 예시 문장이 표시된다

Given 톤을 "realistic"로 설정한 사용자가 일기를 작성했을 때
When AI 댓글이 생성되면
Then 프롬프트에 "현실적이고 담백하게..." 지시가 포함된다
```

### AC-4: 복합 설정 시나리오
```gherkin
Given 사용자가 다음 설정을 완료했을 때:
  - 아바타: 🐱
  - 이름: "나비"
  - 말투: 캐주얼
  - 톤: cheerful
When CompanionScreen을 확인하면
Then 프로필 카드에 🐱 이모지와 "나비" 이름이 표시된다

When 새 일기를 작성하면
Then AI 댓글에 "나비"라는 이름이 포함되고
And 반말로 작성되고
And 밝고 긍정적인 톤으로 생성된다
```

### AC-5: 유효성 검사
```gherkin
Given 사용자가 허용되지 않은 아바타 값을 API로 전송했을 때
When PUT /api/companion/avatar 요청이 처리되면
Then 400 Bad Request 응답이 반환된다
And 에러 메시지는 "유효하지 않은 아바타입니다"이다
```

### AC-6: 기본값 fallback
```gherkin
Given 신규 가입 사용자가 설정을 변경하지 않았을 때
When 첫 일기를 작성하면
Then 기본 아바타(🌰), 포멀 말투, warm 톤으로 AI 댓글이 생성된다
```

## 11. Edge Cases

### 엣지 케이스 1: 아바타 이모지 미지원 디바이스
- **상황**: 일부 안드로이드 구형 디바이스에서 특정 이모지 미지원
- **해결**: fallback 텍스트 표시 (예: "식물", "동물")

### 엣지 케이스 2: 말투 변경 중 AI 댓글 생성
- **상황**: 사용자가 말투 변경 직후 바로 일기 작성
- **해결**: 트랜잭션 순서 보장, 최신 설정 사용

### 엣지 케이스 3: 프롬프트 토큰 길이 초과
- **상황**: 말투 + 톤 + 레벨 설명이 모두 포함되어 토큰 한계 근접
- **해결**: 프롬프트 길이 모니터링, 최대 길이 제한

### 엣지 케이스 4: 잘못된 enum 값 저장
- **상황**: 클라이언트/서버 버전 불일치로 미지원 값 전송
- **해결**: DB constraint + API validation 이중 검증, 기본값 fallback

### 엣지 케이스 5: 동시 다중 설정 변경
- **상황**: 사용자가 빠르게 여러 설정을 연속 변경
- **해결**: optimistic update + 최종 일관성 보장

### 엣지 케이스 6: 아바타 선택 모달에서 뒤로가기
- **상황**: 아바타 선택 후 저장하지 않고 뒤로가기
- **해결**: 변경 사항 무시, 원래 아바타 유지

### 엣지 케이스 7: 성격 톤 "realistic" 선택 시 위기 상황
- **상황**: realistic 톤이 위기 상황에서 부적절할 수 있음
- **해결**: 위기 감지 시 톤 무시하고 안전 우선 응답 생성
